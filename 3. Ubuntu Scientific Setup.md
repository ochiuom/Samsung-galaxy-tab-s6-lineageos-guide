# Ubuntu Scientific Setup — ARM64 / Tablet Workflow

A personal setup guide for scientific computing on Ubuntu (ARM64), including
stylus support, Miniforge/SageMath, ROOT, and a remote-display workflow via
Termux + XFCE on a tablet.

---

## 1. Stylus & Input Tools

```bash
sudo apt install xournalpp
sudo apt install xinput libwacom-bin
```

- **Xournal++** — PDF annotation and handwriting
- **xinput / libwacom** — tablet/stylus input configuration

---

## 2. Miniforge + Conda/Mamba Environment

### 2.1 Install Miniforge (ARM64)

```bash
wget https://github.com/conda-forge/miniforge/releases/download/26.3.2-2/Miniforge3-26.3.2-2-Linux-aarch64.sh
bash Miniforge3-26.3.2-2-Linux-aarch64.sh
source ~/.zshrc
```

### 2.2 Create the `sage-env` environment

```bash
conda create -n sage-env
conda activate sage-env
```

### 2.3 Install mamba into base, then SageMath + ROOT

```bash
conda install -n base -c conda-forge mamba
mamba install -c conda-forge sage
mamba install -c conda-forge root
```

## 5. Termux Remote Display Workflow

### Goal

Run gnuplot / ROOT / SageMath plots over SSH from NixOS, with graphics
rendered in an XFCE desktop on the tablet via Termux.

### 5.1 Set a Static IP on the Tablet

Before anything else, assign a static IP to avoid the SSH target changing
between sessions. Do this in Android Wi-Fi settings (or router DHCP
reservation):

```
IP: 162.108.14.4
```

### 5.2 Start Services in Termux

```bash
# Start SSH daemon
sshd

# Start XFCE desktop (VNC or X server must already be running on :0)
./startxfce4_chrootubuntu.sh
```

### 5.3 SSH into Termux from NixOS

Find Your Termux Username

```bash
whoami
# u0_a157
```

Use this in the SSH command:

```bash
ssh -p 8022 u0_a157@192.168.31.214
```

### 5.4 Forward DISPLAY to the Tablet's X Session

Once logged in via SSH:

```bash
export DISPLAY=:0
```

All subsequent graphical commands (gnuplot, ROOT TCanvas, Sage `plot()`) will
render into the XFCE desktop running on the tablet.

### 5.5 Plotting in Each Tool

**gnuplot** — run a script directly:

```bash
gnuplot raman_si.gp
```

**ROOT + SageMath** — activate the conda env first, then either C++ macro or Python:

```bash
conda activate sage-env

# C++ macro via ROOT interpreter
root FittingDemo.C

# or Python
python3 FittingDemo.py
```

All windows will appear in the XFCE session on the tablet display.

## 6. VS Code + SageMath Jupyter Kernel

### 6.1 Install VS Code (ARM64)

Download the `.deb` from https://code.visualstudio.com/Download# (pick **Linux ARM 64 .deb**)

```bash
chmod 644 ~/Downloads/code_*.deb
sudo apt install ~/Downloads/code_*.deb
```

### 6.2 Open Notebook and Select Kernel

1. Open VS Code
2. File → Open `sage_test.ipynb`
3. Top-right → **Select Kernel** → **Python Environments** → choose `sage-env`

### 6.3 Run the Notebook

Hit **Run All** — plots and fits will render inline, confirming SageMath + ROOT are wired up correctly.
